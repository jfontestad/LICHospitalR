ts_readmit_excess_tbl {LICHospitalR}R Documentation

Time Series - Monthly Readmission Excess Tibble

Description

This takes data from the ts_readmit_excess_query() and makes an internal summary table using timetk::summarise_by_time with the following possible choices: "year", "month", "week", these are checked inside of the function, if something else is chose an error will be thrown and the function will exit. It defaults to "month".

Usage

ts_readmit_excess_tbl(.data, .by_time = "month", .date_col)

Arguments

.data

The data passed in from ts_readmit_excess_query()

.by_time

The choices are "year", "month", "week", defaults to "month"

.date_col

The column containing the date variable of interest

Details

  • Returns a tibble

  • Expects ts_readmit_excess_query() as the data argument

  • Cleans the table names and selects the following columns

  1. dsch_date

  2. dsch (which just equals 1 as a record column)

  3. ra_flag

  4. rr_bench

  • dsch_date gets mutated using lubridate::ymd(dsch_date)

  • The output columns are:

  1. date_col

  2. value - the excess readmit rate.

Value

A tibble

Author(s)

Steven P. Sanderson II, MPH

Examples

## Not run: 
library(healthyR)

data <- ts_monthly_readmit_excess_query()

data %>%
  ts_readmit_excess_tbl(.by_time = "year")

data %>%
  ts_readmit_excess_tbl(.by_time = "month") %>%
  ts_plt(.date_col = dsch_date, .value_col = value)

data %>%
  ts_readmit_excess_tbl()

data %>%
  ts_readmit_excess_tbl(.by_time = "week") %>%
  save_to_excel(.file_name = "weekly_readmit_excess")

## End(Not run)
